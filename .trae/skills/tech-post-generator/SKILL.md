# 技术博客文章生成器

## 功能

此技能用于根据用户提出的技术，生成一篇介绍该技术用法的博客文章（Markdown格式），并自动存放在项目的`_posts`文件夹内。生成的文章将：

- 重点突出技术的实用性和实际用法
- 参考用户已有的tmux文件风格
- 包含清晰的步骤说明和使用示例
- 遵循Jekyll博客文章的格式要求

## 使用场景

当用户需要：
- 为新学习的技术创建一篇实用指南
- 分享技术使用经验和技巧
- 为项目添加技术相关的博客文章
- 参考已有的tmux文件风格创建类似的技术文章

## 使用方法

1. 用户提出需要介绍的技术名称和简要描述
2. 技能会分析用户已有的tmux文件风格
3. 生成一篇符合要求的博客文章
4. 将文章保存到`_posts`文件夹内，文件名遵循Jekyll命名规范（YYYY-MM-DD-技术名称.md）

## 文章结构

生成的文章将包含以下部分：

- **标题**：技术名称
- **日期**：生成当天的日期
- **布局**：single（符合Jekyll要求）
- **分类**：技术相关分类
- **标签**：技术相关标签
- **内容**：
  - 技术简介
  - 安装方法（如果适用）
  - 基本用法
  - 高级技巧
  - 实用示例
  - 总结

## 示例

用户提出要创建关于"Docker"的技术博客文章，技能会：

1. 分析用户已有的tmux文件风格
2. 生成一篇名为`2026-01-17-docker.md`的文件
3. 内容重点介绍Docker的实际用法，如容器创建、管理、部署等实用技巧
4. 将文件保存到`_posts`文件夹内

## 注意事项

- 生成的文章会参考用户已有的tmux文件风格，但会根据不同技术的特点进行适当调整
- 用户可以在生成后对文章进行修改和补充
- 技能会自动处理Jekyll博客文章的格式要求，确保文章可以正常显示

## 输入输出示例

### 输入
```
请为Docker技术创建一篇实用指南博客文章
```

### 输出
```
已生成Docker技术的实用指南博客文章：_posts/2026-01-17-docker.md

文章内容包括：
- Docker简介
- 安装步骤
- 基本命令
- 容器管理技巧
- 实际应用示例
- 常见问题解决方案
```
